# 📄 PRD — 端侧离线语音对话助手 v1

## 1. 产品概述（Product Overview）

### 1.1 产品定位

**离线优先的端侧语音对话系统（Offline-First On-device Voice Assistant）**

在现代 Android 手机上，通过 CPU 推理实现稳定、可控、可测的语音理解与文本生成能力，不依赖云端即可运行，并支持可选联网增强。

---

### 1.2 核心价值主张

相对于云端 AI 助手，本产品的差异化价值：

* ✅ 离线可用（无网 / 弱网可运行）
* ✅ 确定性时延（可预测响应）
* ✅ 隐私闭环（语音与文本不离开设备）
* ✅ 零云推理成本
* ✅ 可观测资源占用
* ✅ 可集成设备本地能力（未来扩展）

---

## 2. 目标用户与使用场景

### 2.1 目标用户

* 技术爱好者 / 开发者
* 注重隐私的用户
* 离线环境使用者
* 研究或评估端侧 AI 能力的团队

---

### 2.2 核心使用场景

* 飞行模式下的语音问答
* 无网络环境的信息查询
* 本地语音交互实验
* Edge AI 评估与演示

---

## 3. 目标设备与技术假设

### 3.1 目标硬件平台

| 项目  | 规格                          |
| --- | --------------------------- |
| CPU | Snapdragon 8 Gen1 / Gen2 级别 |
| RAM | 8–12 GB                     |
| 架构  | ARM64-v8a                   |
| 加速  | CPU 推理（不依赖 GPU/NPU）         |
| 存储  | ≥ 5 GB 可用空间                 |
| 系统  | Android 10+                 |

---

### 3.2 运行约束

* 单设备运行
* 不依赖 Google 服务
* 不要求 root
* 支持长期运行稳定性

---

## 4. 支持语言

* 🎯 v1：仅中文普通话（Mandarin Chinese）
* ASR 输出语言：简体中文
* LLM 回复语言：简体中文

---

## 5. 交互模型

### 5.1 输入方式

* Push-to-Talk（按住说话）
* 松开结束录音

---

### 5.2 对话模式

* 单轮问答（默认）
* 可选最近 N 轮上下文（默认关闭）

上下文策略：

* 仅保留最近 N 轮文本
* 可清除
* 可配置最大 token 数

---

### 5.3 输出方式

* 文本流式输出（token streaming）
* 不包含 TTS（v1 不支持）

---

## 6. 功能范围（Scope）

## 6.1 v1 核心功能（必须实现）

### 🎤 语音 → 文本 → 回复完整链路

```
录音 → ASR → Prompt 构建 → LLM → 流式文本输出
```

---

### 📊 系统状态机

应用必须可观测状态：

* Idle
* Recording
* Transcribing
* Thinking
* Streaming
* Done
* Error

---

### ⚡ 性能观测

每次对话记录：

* 音频时长
* ASR 耗时
* LLM 首 token 延迟
* 总生成时间
* 生成 token 数
* 峰值内存
* CPU 使用率（可选）

---

### 📦 模型管理

* 支持本地模型加载
* 支持外部导入（adb / 文件复制）
* 模型版本与校验（建议 sha256）
* 加载失败可恢复

---

### 🔄 流式输出

* token 实时显示
* 不等待完整结果

---

### ❌ 错误处理与降级

典型错误：

* 模型不存在
* 内存不足
* 推理失败
* 音频读取失败
* 权限问题

降级策略示例：

* 减少线程数
* 降低上下文长度
* 提示用户关闭其他应用

---

## 6.2 v1 可选联网增强（Cloud-Augment）

默认行为：完全端侧推理

联网增强触发条件：

* 用户开启
* 请求复杂或失败
* 用户显式选择

可选增强功能：

* 云端 LLM 推理
* 网络检索
* 长文本处理

必须保证：

* 联网功能不影响离线能力
* 可随时关闭

---

## 6.3 v1 不包含功能（Out of Scope）

* TTS（语音合成）
* 自动 VAD（持续监听）
* 多语言支持
* 复杂工具生态
* 多用户系统
* 后台常驻助手
* 智能设备控制

---

## 7. 性能指标（Engineering Targets）

### 7.1 离线能力

* 飞行模式可完整运行

---

### 7.2 时延目标（旗舰设备）

| 阶段          | 目标           |
| ----------- | ------------ |
| ASR 完成      | ≤ 2 s（5 秒语音） |
| LLM 首 token | ≤ 1 s        |
| 总体响应        | ≤ 5 s（短回答）   |

---

### 7.3 稳定性

* 连续运行 ≥ 30 次不崩溃
* 无内存泄漏
* 无明显性能退化

---

### 7.4 资源占用

* 峰值内存可观测
* CPU 使用率可观测
* 不导致系统严重卡顿

---

## 8. 音频与语音处理要求

* 单声道 PCM 输入
* 16 kHz 推荐采样率
* 录音完整无丢帧
* 支持录音权限管理

---

## 9. 安全与隐私

* 音频与文本默认仅本地处理
* 不自动上传数据
* 本地日志可关闭
* 用户可清除历史

---

## 10. UX 要求

### 10.1 状态反馈

UI 必须清晰显示：

* 当前状态
* 推理阶段
* 进度或动画

---

### 10.2 首次启动

可能存在：

* 模型加载时间较长
* 需提供加载提示

---

### 10.3 错误提示

必须可理解、可恢复

---

## 11. 技术架构（概览）

### 11.1 端侧推理栈

ASR：

* Whisper.cpp
* CPU 推理

LLM：

* llama.cpp
* Qwen2.5-3B-Instruct（量化）

---

### 11.2 Android 架构

```
UI (Kotlin)
   ↓
Runtime Layer
   ↓
JNI Bridge
   ↓
Native Engine (C++)
   ↓
Whisper / Llama
```

---

## 13. 成功标准（Success Criteria）

v1 被认为成功，当：

* 离线模式可稳定运行
* 语音链路完整可用
* 性能指标达标
* 用户可理解状态与结果
* 系统可长期运行无崩溃
